## 1. 目标收敛

### 1.1 终极目标

> **让 InCorp 新加坡会计团队的月结前端处理从"大量邮件手工处理+凭证卡在邮件系统+月结催票靠人工翻邮件"转变为"AI 自动处理邮件凭证→实时入 Xero+智能历史邮件检索催票"——核心是 AI 接管邮件侧的凭证捕获和检索，让会计经理从每天大量查邮件发邮件中解放出来，聚焦月结质量和客户价值沟通。**
>
> **★ 会计经理直接确认的 M8 工作流：** "会计经理每天会收到大量不同公司客户侧发送的邮件，里面会附加各种 invoice、银行流水等文件，然后需要再手动导入 file.ai 系统中进行各种格式文件（包括 invoice、银行流水）的结构化信息提取，然后经人工确认后，会自动提交到 Xero 系统（财务记账系统）"
>
> **★ 会计经理直接确认的 M9 工作流：** "在月结时，经常发现有些 invoice 缺失，又需要查询大量与该公司客户的历史邮件，并发邮件给对方进行催票。在这些场景中，需要大量查邮件、发邮件的工作，比较费时费力"
>
> **★ M8 和 M9 均直接来自会计经理 2026-04-20 一线反馈，是本次 SRP v2.0 核心推导依据。**

## 7. 反向反馈机制

### 7.1 事件源

| 事件类型 | 来源 | 采集方式 | 频率 |
|---------|------|---------|------|
| runtime_logs | Agent 运行日志 | 系统日志收集 | 实时 |
| user_ratings | 会计专员对 Agent 输出的评分 | 应用内点赞/点踩 | 按需 |
| user_comments | 会计专员评论/修改记录 | Audit Layer 记录 | 实时 |
| exception_events | FUSE 触发 / 异常告警 | 监控系统 | 实时 |
| kpi_deltas | A-01~A-07 实际值偏离目标 | KPI 监控看板 | 每日 |

### 7.2 分类引擎规则

| 触发条件 | 类型 | 动作 |
|---------|------|------|
| 置信度 < 0.6 且命中次数 > 10 | structural_change | 生成 DCR → 更新 SRP → 重新打造 Agent |
| 命中率 < 70% 且为高优先级场景 | structural_change | 生成 DCR → 更新 SRP → 重新打造 Agent |
| 用户反馈 = 微调（改词/改格式） | hot_update | 直接改 Agent 配置，不回灌 SRP |
| 用户反馈 = 业务逻辑错误 | structural_change | 生成 DCR |
| KPI 实际值偏离目标 > 20% | structural_change | 生成 DCR，分析根因 |
| 异常事件频率 > 5%/天 | structural_change | 暂停 Agent，生成 DCR |

### 7.3 DCR 流程

```
异常/反馈事件
      │
      ▼
Audit Layer 分类
      │
      ├─▶ hot_update：直接修改 Agent 配置
      │
      └─▶ structural_change：生成 DCR
                │
                ▼
          FDE + 客户产品 共同审批
                │
                ├─▶ 批准：更新 SRP 版本 → 重新打造 Agent
                │
                └─▶ 拒绝：记录原因 → 调整期望
```

### 7.4 Governance

| 要素 | 内容 |
|------|------|
| **DCR 审批链** | FDE + 客户产品（role_b3_account_manager） |
| **SRP 版本控制** | git + version tag（v2.0 → v2.1 → ...） |
